ダッシュボードが本番環境になったりlocalhost環境になったりする

本番環境のDBをlocalhost側にインポートしたとき、 ダッシュボードが本番環境になってしまうことがある。

原因

wp_optionsテーブルのsiteurlとhomeのURLが本番環境やlocalhostのURLのままになっている。

修正

DBのwp_optionsテーブルを開き、option_nameフィールドがsiteurlとhomeになっている行のoption_valueを正しいURLに書き換える。


その他の対応方法

wp-config.phpの先頭あたりに以下の定数を追記する修正方法もある。 
	// localhost開発用のコード
	define ('WP_SITEURL', 'http://localhost/○○/wp/');
	define ('WP_HOME', 'http://localhost/○○/wp/');

タグについて

タグ登録画面の表示

左メニュー →　投稿→　タグ

タグの新規追加

「名前」にはタグの名前を。日本語などの全角文字を入力。
スラッグはURLに埋め込む文字。半角英数字が望ましい。

投稿記事にタグを付加する。

どれでも良いのでブロックを一つ選択→オプションをクリック→「追加設定を表示」をクリック→　右側に設定ウィンドウが表示されるので「投稿」タブを選択→ 「新規タグを追加」でタグ入力ができる。


「新規タグの追加」に入力できるものは登録済みのタグの他、未登録のタグも新規で追加入力できる。 必須というわけではないが、後で新規追加したタグのスラッグ(slug)をタグ登録画面で変更しておいてもいい。

データベース中におけるタグデータの保存場所

タグはDBのwp_termsテーブルに保存される。

wp_termsテーブルのフィールド
term_id name slug term_group
