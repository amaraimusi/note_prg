Windows10でLINUXを使えるようにする

  1. スタートメニューを右クリック
  2. 左メニューの「アプリと機能」
  3. 右上の「プログラムと機能」
  4. 左側の「Windowsの機能の有効化または無効化」
  5. 「Windows Subsystem for Linux(Beta)」にチェックを入れる
  6. PCを再起動
  7. スタートメニュー　→　設定
  8. 更新とセキュリティ
  9. 左メニューの「開発者向け」
  10. 「開発者モード」を選択
  11. コマンドプロントを立ち上げる
  12. bashコマンドを実行　例→ C:\ >bash
  13. メッセージを読みながらyを選択。（反応が悪いので注意。少し待つかEnterキーで回復するっぽい)
  14. ユーザー名を入力
  15. 続いてパスワードを入力（画面には入力が表示されてないが、入力されている。
  16. 再パスワードを入力
  17. 以上でOK。
    次回からはコマンドプロントを起動し、「bash」と入力するだけで利用できるようになる。

Linuxの基本コマンド

コマンド名説明例
$ lsカレントディレクトリ内のファイルやフォルダを一覧表示します。$ ls
$ cdカレントディレクトリを移動する。$ cd \c
$ cd ~ホームに戻る。

$ shシェルを実行する$ sh test.sh
$ yum list installedLinuxにインストールされているプログラムを一覧表示するhttpで絞り込み→$ yum list installed | grep http
$ ps alxw動作中のプログラムを一覧表示する
$ ss開いているポート番号を一覧表示する

一般ユーザー($)とrootユーザー(#)

$は一般ユーザーであることを表している。#はrootユーザーを表している。

VSFTPDのインストールとコマンド

CentOS7で検証

インストール

  1. VSFTPDインストール 
    # yum install -y vsftpd
  2. FTPコマンドをインストール 
    # yum install -y ftp
  3. サービスをスタート 
    # service vsftpd start

コマンド

VSFTPDがインストールされているか確認する
#rpm -qa | grep vsftpd

サービスの状態を確認
#systemctl status vsftpd.service

サービスを止める
# service vsftpd stop

サービス再起動
# service vsftpd restart

自動起動設定
# systemctl enable vsftpd.service

権限を与える
# sed -i 's/root/#root/g' /etc/vsftpd/user_list /etc/vsftpd/ftpusers

2018-2-10

catコマンド | ファイル内容の閲覧、別ファイルに出力

neko/test.txtファイルの内容を見る 
	$ cat neko/test.txt
※長文を見るときはlessコマンドを使うよい

test.txtの内容をtest2.txtに書き出す（test2.txtに上書き） 
	$ cat test.txt > test2.txt

test.txtの内容をtest2.txtの末尾に追加する（上書きにあらず） 
	$ cat test.txt >> test2.txt

test.txtとtest2.txtの中身を連結して出力する。（出力のみで保存はされず） 
	$ cat test.txt test2.txt

rmコマンド | ファイルやディレクトリを削除する

ファイルを削除する（xxx.txtファイルを削除する） 
	$ rm xxx.txt

ディレクトリを中身のファイルごと削除する 
	$ rm -r testDir

拡張子がtxtのファイルを削除する 
	$ rm *.txt

【 rm 】 ファイルやディレクトリを削除する
2018-2-10

tarコマンド | 圧縮と解凍

圧縮

test2ディレクトリをtest2.tar.gzファイルとして圧縮する。
事前にcdコマンドでtest2が存在するディレクトリに移動しておくこと。
	$ tar cvzf test2.tar.gz test2
test2.tar.gzファイルが同ディレクトリに作成される。

解凍

$ tar vxzf test2.tar.gz
